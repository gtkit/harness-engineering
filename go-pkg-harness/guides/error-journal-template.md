## 2026-04-09: [编造 API]

**错误描述**：使用了 `gtkit/httputil.SendJSON()` 函数，该函数不存在
**根因分析**：对 gtkit 库的 API 记忆不准确，没有先确认就直接使用
**正确做法**：不确定 gtkit 下某个包的 API 时，先用本机 `go doc` / `go list` / 模块缓存或官方源码核验；仍无法确认时再请用户确认或提供 go doc 输出
**受影响范围**：所有涉及 gtkit 库的任务

## 2026-04-09: [跳过并发检查]

**错误描述**：在库内部用 map 做本地缓存，没有加锁，多个 goroutine 并发读写导致 panic
**根因分析**：只关注了功能正确性，忽略了库类型被多 goroutine 复用的场景
**正确做法**：声明 concurrent-safe 的类型必须用 sync.RWMutex、sync.Map 或不可变快照保护共享状态；写完代码后执行 `go test -race` 验证
**受影响范围**：所有涉及共享状态的导出类型、缓存、池和后台协程
