# Review Standard

## 输出顺序

1. Findings
2. Open Questions
3. 风险摘要
4. 历史错误对照检查
5. 合规检查摘要

## Findings 规则

1. 先高后低
2. 必须标严重性
3. 必须指出证据
4. 必须说明风险
5. 必须给修复方向

## 严重性

- P0：上线阻断
- P1：合并前修复
- P2：应尽快修复
- P3：优化项

## 必查维度

1. 架构边界
2. 所有权与资源
3. 并发与关闭路径
4. Linux 运行路径
5. AI 运行时安全
6. 测试缺口
7. 发布风险
8. 观测缺口

## 不允许的审查结论

1. “看起来没问题”
2. “应该可以”
3. “大概率没风险”
4. “样例过了所以没事”
