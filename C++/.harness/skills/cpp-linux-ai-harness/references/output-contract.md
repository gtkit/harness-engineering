# Output Contract

## 总规则

1. 严格按用户要求的结构输出
2. 不增加额外章节
3. 不自由发挥
4. 不扩展无关内容
5. 信息不足时只输出“缺失信息清单”

## 实施类任务结构

1. 需求理解
2. 关键信息
3. 变更内容
4. 验证结果
5. 历史错误对照检查
6. 合规检查摘要

## 审查类任务结构

1. Findings
2. Open Questions
3. 风险摘要
4. 历史错误对照检查
5. 合规检查摘要

## 模板类任务结构

1. 需求理解
2. 关键信息
3. 目录树
4. 文件正文
5. 验证矩阵
6. 合规检查摘要

## 信息不足结构

1. 需求理解
2. 缺失信息清单
3. 风险说明
4. 合规检查摘要
