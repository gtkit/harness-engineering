# Error Memory

## 目标

让同类问题在后续任务中默认先被看见、被对照、被防范，而不是重复出现后再补救。

## 触发前动作

1. 读取 `.harness/error-journal.md`
2. 提取：
   - 高风险区域
   - 重复问题
   - 未关闭条目
   - 已定义预防规则

## 追加记录触发条件

1. 命令失败
2. 测试失败
3. 用户纠正
4. 审查发现缺陷
5. 回归问题
6. 误判
7. 同类错误再次出现

## 每条记录必须包含

1. Summary
2. What Happened
3. Root Cause
4. Corrective Action
5. Prevention Rule
6. Related Files
7. Validation Added
8. Linked History

## 结束前动作

1. 对照历史条目检查是否命中同类问题
2. 若命中：
   - 引用旧条目 ID
   - 说明本次新增防护
3. 若未命中但本次出现新问题：
   - 新增条目
4. 没有做对照检查，不得声称“完成”

## 最低预防升级规则

同类问题再次出现时，至少新增一项：

1. 新回归测试
2. 新审查项
3. 新构建门槛
4. 更清晰模块边界
5. 更严格资源约束
