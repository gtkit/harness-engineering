# Workflow

## 固定步骤

1. 读取 `.harness/error-journal.md`
2. 分类任务
3. 加载必要参考文件
4. 按 Logic 四步执行
5. 如出现失败或误判，追加错误日志
6. 输出前执行历史错误对照检查
7. 给出合规检查摘要

## Logic

### 1. 理解需求
- 识别任务边界
- 识别交付物
- 识别生产风险

### 2. 提取关键信息
- 提取已知事实
- 标注缺失信息
- 标注受影响模块
- 标注验证范围

### 3. 按结构组织内容
- 只使用允许的输出模板
- 不自由发挥
- 不混写无关项

### 4. 检查合规性
- 对照 Harness 规则
- 对照验证矩阵
- 对照审查清单
- 对照错误日志

## 结束门槛

只有在以下条件全部满足时，才能给出“完成”类结论：

1. 关键事实明确
2. 结构符合要求
3. 验证证据存在
4. 历史错误已对照
5. 新错误已记录
