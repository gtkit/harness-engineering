# Hidden Risk Checklist

## 内存与资源

- [ ] 所有 FD 都有明确关闭路径
- [ ] 所有线程都有明确结束路径
- [ ] 所有模型上下文都可显式释放
- [ ] 重复加载 / 卸载不会累积泄漏
- [ ] 错误路径不会跳过资源释放
- [ ] 超时路径不会遗留后台工作

## 并发与时序

- [ ] 锁顺序固定
- [ ] 停止信号与工作线程时序已验证
- [ ] 回调不会访问已释放对象
- [ ] 共享缓存有一致性策略
- [ ] 队列上限已验证
- [ ] 取消操作不会导致悬挂

## Linux 运行环境

- [ ] 路径、权限、用户、工作目录未硬编码
- [ ] 启动依赖缺失时能够失败退出
- [ ] systemd / 容器 / 裸机假设明确
- [ ] 时间源使用正确
- [ ] 日志轮转或输出路径已考虑
- [ ] 崩溃后可定位版本与构建信息

## AI 运行时

- [ ] 模型版本与代码版本可关联
- [ ] 模型损坏时不会进入服务
- [ ] 设备不可用时行为明确
- [ ] 退化路径不会悄悄改变结果语义
- [ ] 批处理与并发策略不冲突
- [ ] 大输入、长输入、异常输入有拒绝策略

## 安全与合规

- [ ] 不记录敏感输入原文
- [ ] 外部模型 / 插件来源可信策略明确
- [ ] 配置中的凭据不会进入日志
- [ ] 外部命令执行最小化
- [ ] 依赖版本与来源可追溯
- [ ] 失败场景不会泄露内部细节
