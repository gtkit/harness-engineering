# Regression Policy

## 回归分类

1. build regression
   - 以前可编译，现在不可编译
2. correctness regression
   - 结果错误、行为改变、协议不兼容
3. performance regression
   - 延迟、吞吐、启动时间、资源占用恶化
4. memory regression
   - 泄漏、UAF、越界、释放顺序错误
5. concurrency regression
   - 竞态、死锁、饥饿、关闭悬挂
6. ops regression
   - 启动、停止、健康检查、部署失败
7. ai regression
   - 模型加载失败、推理错误、版本路由错误、退化失效

## 阻断规则

以下任一情况为阻断项：

1. P0 / P1 缺陷
2. 必需验证矩阵失败
3. Sanitizer 失败
4. 启动 / 停止 / 健康检查失败
5. AI 加载或超时路径失败
6. 无法解释的性能显著下降
7. 历史同类错误再次出现且无新增防护

## 修复要求

1. 每次回归修复必须新增回归验证
2. 修复必须回链 `.harness/error-journal.md`
3. 同类回归再次出现时，必须新增更强防护：
   - 更强测试
   - 更强审查
   - 更清晰边界
   - 更严格构建门槛

## 豁免规则

1. 所有豁免必须有：
   - 责任人
   - 风险说明
   - 到期时间
   - 替代控制措施
2. 无期限豁免禁止进入发布路径

## 关闭条件

1. 缺陷复现已确认
2. 修复已验证
3. 回归测试已新增
4. 错误日志已更新
5. 风险说明已关闭或转交
