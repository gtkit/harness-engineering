# Linux Systems Guide

## 适用范围

本文件适用于 Linux 守护进程、CLI、系统服务、设备接入程序、AI 推理服务、资源受限本地推理程序。

## 进程契约

生产可执行程序至少支持以下能力：

1. `--config`
2. `--version`
3. `--dry-run`
4. `--health-check`

如项目存在额外运行参数，统一文档化；未定义参数不得默认接受。

## 启动要求

1. 启动阶段必须完成配置校验
2. 启动阶段必须完成依赖探测
3. 启动阶段必须完成日志与指标初始化
4. 启动阶段必须完成模型或外部依赖可用性检查
5. 关键依赖不可用时，启动必须失败
6. 不允许带病启动

## 信号与退出

1. 明确处理 `SIGTERM`、`SIGINT`
2. 如使用重载配置，明确处理 `SIGHUP` 是否启用；未定义则禁用
3. 收到退出信号后：
   - 停止接收新请求
   - 通知工作线程停止
   - 等待在途任务按超时策略收敛
   - 释放模型与系统资源
4. 退出必须可验证，不得长期挂起

## 文件系统与路径

1. 配置路径、数据路径、缓存路径、日志路径必须配置化
2. 应用名、安装路径、运行用户、目录布局：`需用户补充`
3. 不允许硬编码临时路径、模型路径、设备节点路径
4. 文件写入必须考虑原子性、权限、磁盘满、部分写入

## 权限与最小授权

1. 默认非 root 运行
2. 仅在明确必要时提升权限
3. 使用最小文件权限和最小进程能力
4. 不允许把凭据写入日志
5. 外部调用、模型文件、配置文件必须做权限审查

## FD / Socket / IPC

1. 所有 FD 必须有明确所有者
2. 所有 socket 必须配置超时与错误处理
3. 阻塞 IO 必须说明原因和隔离策略
4. IPC 通道必须有关闭与清理策略
5. 端口、路径、队列名称不得硬编码进生产代码

## systemd 约束

如采用 systemd 部署，至少满足以下要求：

1. 明确 `Restart` 策略
2. 明确启动超时与停止超时
3. 明确运行用户与工作目录
4. 明确日志接入方式
5. 明确依赖的网络、挂载、设备条件
6. 明确资源限制
7. Unit 文件内容与部署路径：`需用户补充`

## 资源控制

1. CPU、内存、文件句柄、线程数、队列深度必须有预算
2. 如存在 GPU，显存预算与设备隔离策略必须明确
3. 超出预算时必须进入退化路径或拒绝新任务
4. 不允许在资源失控时继续放大负载

## 时间与时钟

1. 统一使用单调时钟做超时与延迟计算
2. 墙钟仅用于日志和审计
3. 定时任务必须处理漂移与重入

## 崩溃与恢复

1. 崩溃必须留下足够诊断信息
2. 生产构建必须保留可追溯符号策略
3. 重启后必须避免损坏状态继续传播
4. 恢复策略必须区分：
   - 自动恢复
   - 人工介入
   - 禁止恢复

## 安全要求

1. 不信任环境变量、命令行、配置文件、外部输入
2. 模型文件、插件、动态库、脚本如可替换，必须有校验策略
3. 外部命令执行必须最小化
4. 不允许无边界读取本地文件、设备、网络内容

## 验证要求

1. 启动 / 停止路径必须有自动化验证
2. 信号退出必须有自动化验证
3. 异常配置必须有失败验证
4. 文件权限错误必须有失败验证
5. 资源耗尽路径必须有演练或模拟验证
