# C++ Coding Guide

## 语言基线

1. 默认支持 `C++20`
2. 如工具链矩阵全部通过，可启用 `C++23`
3. 语言标准必须在构建系统中显式声明
4. 禁止混用不受支持的编译器扩展

## 编译质量门槛

1. 默认开启高等级告警
2. CI 中告警按错误处理
3. 最低要求覆盖：
   - 未初始化
   - 隐式窄化
   - 符号转换
   - 覆盖遗漏
   - 虚析构问题
   - 可疑复制 / 移动语义
4. 编译通过不是质量通过，必须叠加静态检查和测试

## 头文件规则

1. 头文件最小暴露
2. 不在公共头文件中包含重量级依赖，能前置声明则前置声明
3. 头文件必须自包含
4. 禁止跨模块包含私有头文件
5. 公共接口优先使用稳定、清晰的值语义或轻量引用语义

## 所有权与资源管理

1. 一切资源通过 RAII 管理
2. 生产代码中禁止裸 `new/delete`
3. 文件描述符、线程、锁、socket、显存句柄、模型句柄必须封装成可审查的所有权对象
4. `std::unique_ptr` 表示唯一所有权
5. `std::shared_ptr` 仅在确有共享生命周期时使用
6. 不允许以“方便”为理由扩大对象共享范围
7. 资源释放必须可验证

## 接口设计

1. 接口优先表达清晰语义，不拼装难以审查的布尔参数组合
2. 参数超过 4 个时，优先使用命名配置结构
3. 输出结果必须清楚区分：
   - 成功值
   - 可恢复失败
   - 不可恢复失败
4. C++23 项目优先使用 `std::expected`
5. C++20 项目使用项目内 `Result/Status` 类型
6. 未定义错误语义的接口不得进入主干

## 错误处理

1. 跨模块边界使用显式错误返回
2. 不允许跨线程边界抛异常
3. 不允许跨 C ABI / 插件 ABI 抛异常
4. 初始化失败必须中止启动
5. 运行期可恢复错误必须：
   - 记录上下文
   - 更新指标
   - 返回明确状态
6. 禁止空 catch、禁止吞错、禁止只打印不传播

## 并发规则

1. 禁止脱管线程
2. 线程启动必须绑定生命周期所有者
3. 停止流程必须支持显式取消
4. 共享状态必须声明同步策略
5. 锁顺序必须固定
6. 禁止在锁内执行长耗时 IO 或推理
7. 原子变量只能解决单变量同步，不能替代状态机设计
8. 不允许在无测试与无 TSan 的前提下宣称线程安全

## 容器与内存

1. 优先使用标准容器
2. 传递只读连续区间优先使用 `std::span`
3. 字符串只读视图优先使用 `std::string_view`
4. 对象缓存、对象池、内存池必须给出收益证据
5. 不允许“先优化再验证”
6. 大对象复制必须可解释

## 日志与错误上下文

1. 日志必须结构化
2. 日志中不得泄露密钥、令牌、敏感路径、模型私有数据
3. 错误日志至少包含：
   - 模块
   - 操作
   - 关键参数摘要
   - 错误码或状态
4. 高频路径必须控制日志量

## 测试要求

1. 每个缺陷修复必须新增回归测试
2. 每个并发模块必须有并发测试或压力测试
3. 每个资源敏感模块必须有释放验证
4. 公共工具库必须具备独立单元测试
5. 模型适配器必须具备失败路径测试

## 禁止事项

1. 全局可变单例
2. 隐式共享可变状态
3. 未经验证的微优化
4. 以宏代替类型安全接口
5. 以原始指针表达所有权
6. 在头文件中放复杂实现且缺乏边界控制
7. 把业务语义塞进通用工具库
